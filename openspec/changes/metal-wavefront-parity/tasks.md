## 3. Wavefront path integrator on Metal (design phase 3)

- [x] 3.1 Implement the Metal adapter: encode generate → intersect → build_args → scatter → per-material shade → resolve into one frame command encoder with barriers; per-material shade via `dispatch_indirect` (or fallback). *(`metal_wavefront._MetalPathRecorder` drives the shared `record_path_loop` over one `MetalFrameEncoder`/frame; `MetalFrameEncoder` gained `uniforms=` byte-blob blocks (the `wfTile` push-constant analogue, set via dereferenced `set_data` — D4), `bindless=` array binding, and `clear()` (the `vkCmdFillBuffer` analogue). Shade uses native indirect when probed, else the live CPU-readback fallback: `flush()` → read the `(x,y,z)` triple → equal-count direct dispatch (zero-count queues skip).)*
- [x] 3.2 Construct `WavefrontPathPass` on a `MetalContext` in `renderer.py` (lift the Vulkan-only gate); confirm the wavefront env camera-ray pass replaces the megakernel dispatch. *(`_ensure_wavefront_path_pass_metal` builds the Metal pass lazily (no megakernel compile in wavefront mode — `scene_bindings_only`); `execution_modes` now lists Wavefront on Metal; BDPT+wavefront falls back to megakernel on Metal until phase 4 (`effective_execution_mode_index`). New `_msl_layout_source` points the MSL relocators (fc / graph params / std-surface / mtlx-skin) at the wavefront pass's reflected layouts, with a post-build re-upload of graph + flat-material buffers (the scene built them before any Metal reflection existed). `_render_scene_metal` gates wavefront vs megakernel for both headless + windowed; `read_accumulation` gained the Metal readback branch.)*
- [x] 3.3 Compile the wavefront `.slang` modules through the Metal in-process Slang→Metal path; resolve any scalar-layout/`float3` mismatch surfaced by task 1.5. *(Per-entry link of `wavefront_path.slang` in one slang session (same compiler surface as the megakernel: `SKINNY_METAL=1`, column-major, MSL layout). Three Metal-target shader gates were required: (1) **31-buffer-slot cap** — the wavefront program's scene+queue buffer globals overflow Metal's argument table, so the wavefront-native record drain (`wf_records.slang` → stubs, drops wfRecStack/wfRecCount/recordBuf/recordCounter) and the neural-proposal weight buffers (`neural_proposal.slang` → pdf-0 stubs, phase 6 must revisit) are compiled out under `SKINNY_METAL`; (2) **bindless-array slot aliasing** — Slang's Metal backend emits the texture *array* parameter without an explicit `[[texture(N)]]` attribute, so Metal auto-assigns it from 0 while slang-rhi binds at the reflected (post-`outputBuffer`/`accumBuffer`/`hudMask`) indices; in kernels that dead-strip those preceding textures every bindless sample aliased the wrong texture (wood sampled `outputBuffer` → rendered gray). Fixed by declaring `flatMaterialTextures` first on the Metal branch of `bindings.slang` so reflected base == auto-assigned base 0 in every kernel. (3) `wfState` reflected stride asserted == `path_state_size(msl=True)`. Vulkan SPIR-V verified byte-identical (`mainImage` + `wfPathShadeFlat` recompiles `cmp`-equal to HEAD).)*
- [x] 3.4 Headless A/B: render the test scene with the wavefront path integrator on Metal vs Vulkan; assert structural-exact equality and converged color within the megakernel perceptual tolerance. *(`tests/test_metal_wavefront_path_ab.py` (gate `RUN_METAL_WAVEFRONT_COMPILE=1` under `scripts/guarded_metal.sh`): three-materials demo, 96 samples — Metal vs Vulkan wavefront accumulation is **bit-identical** on this host (cross-match 1.0000, rel-MSE 0.00000, corr 1.00000; both backends end on the same Metal GPU), beating the noise-floor criterion and the megakernel perceptual tolerances. The structural-AOV channel needs the megakernel tool dispatch (not built in wavefront mode), so geometric/ID parity is pinned by the archived megakernel structural test (same `traceScene`); this A/B pins the staged estimator on top via the noise-floor criterion. Regressions green: Metal megakernel procedural parity, Vulkan wavefront A/B (5/5), foundation+driver+layout (32/32), ruff.)*
