## Why

`docs/ReSTIR.md` and `docs/NeuralGuiding.md` each present their math as
rendered-SVG equations, a one-line `> Implements:` pointer per equation, and a
global "Equation → implementation map" table. None of the three shows the actual
**arithmetic** that realizes an equation, and none binds a math symbol to the
**slang identifier** that carries it. A reader who wants to go from `W =
Σwᵢ/(M·p̂)` or `y = y_k + h_k(sθ²+…)/(…)` to the line of Slang doing it must open
the shader, find the function named in the table, and re-derive the
correspondence by hand. The shaders already self-document part of this (`float
wSum; // Σ RIS weights`, `float pHat; // cached target p̂(y)`), so the binding
exists in code — it is just not surfaced beside the equation.

This change closes that gap for both reference docs, and does it with a generator
so the embedded code cannot silently drift from the shaders it is copied from.

## What Changes

- **New doc-build tool `docs/diagrams/embed_code.cjs`** (Node, alongside the
  existing `eqsvg.cjs` / equation-render scripts) — slices marked regions out of
  the Slang sources and fills marked placeholders in the Markdown:
  - **Slang markers** — `// DOC:<key>#<slice> start` / `… end` comment pairs wrap
    the snippet regions. A snippet may name several slices (`sig`, `core`) so a
    "signature + key line" excerpt of a large function is assembled from
    non-adjacent slices joined by an elision (`    // …`). Markers are comments,
    so no shader semantics and **no `.spv` recompile** are required.
  - **Markdown placeholders** — `<!-- CODE:<key> <slice,slice> -->` … `<!-- /CODE:<key> -->`
    regions in the docs; the generator replaces the body with a ```slang fenced
    block tagged with its `// from <file>.slang` provenance.
  - **Orphan check** — verifies every `CODE:` placeholder resolves to slang
    markers and (optionally) every `DOC:` marker is referenced by a doc, failing
    loudly so a removed/renamed function is caught.
- **`docs/ReSTIR.md`** — under each equation (§1–§7), a generated signature+key-line
  Slang snippet and a per-equation symbol→code table.
- **`docs/NeuralGuiding.md`** — same, for §1–§11.
- **Slang markers** — `// DOC:` comment pairs added to the functions already named
  in each doc's implementation map: `restir/reservoir.slang`,
  `restir/light_ris.slang`, `restir/restir_primary.slang`, `sampling/reuse.slang`
  (ReSTIR); `sampling/neural_flow.slang`, `sampling/neural_proposal.slang`,
  `sampling/proposal.slang`, `integrators/path_record.slang` (Neural).
- The existing per-equation `> Implements:` pointers and the global
  "Equation → implementation map" tables stay as the index; the new blocks are
  the zoom-in.
- No code path, shader semantics, parameter, or behaviour change.

## Capabilities

### New Capabilities
- `docs-equation-code-embedding`: a marker-driven generator that embeds excerpts
  of the Slang sources into the reference docs and keeps them from drifting.

### Modified Capabilities
- `restir-di`: the reference-doc requirement gains that each documented equation
  be accompanied by the generated Slang excerpt that implements it and a
  symbol→code mapping.
- `neural-directional-proposal`: same addition for `docs/NeuralGuiding.md`.

## Impact

- **Docs (new):** `docs/diagrams/embed_code.cjs` (+ its README/usage note).
- **Docs (edit):** `docs/ReSTIR.md`, `docs/NeuralGuiding.md` (CODE placeholders +
  symbol tables); a pointer in `CLAUDE.md`'s docs-upkeep section that these two
  docs carry generated code blocks regenerated by `embed_code.cjs`.
- **Shaders (edit):** comment-only `// DOC:` markers in the eight slang files
  above. No semantic change, no `.spv` recompile.
- **Code:** none beyond the Node generator. No test or runtime behaviour change.
- **Source of truth:** the shipped shaders remain authoritative; the doc snippets
  are generated copies and MUST be regenerated, not hand-edited.
