- [x] 1.1 Spike: link the full `main_pass.slang` (all included modules, entry `mainImage`) in-process on `-target metal` via SlangPy. Record whether it links clean or needs a `.metallib` shell-out fallback (D2). Note any Vulkan-only construct that needs a Metal-target branch (O2).
  - **Outcome (O1 RESOLVED — in-process, no `.metallib` shell-out):** SlangPy `create_device(type=metal, include_paths=[shaders, mtlx/genslang])` → `load_module_from_source` → `link_program(mainImage)` → `create_compute_kernel` runs the in-process Slang→Metal compiler. **No Metal-target language rejection surfaced** — `[[vk::binding(N)]]` attributes, `RWTexture2D`, `StructuredBuffer`, generics/`IMaterial` all map cleanly; `mainImage` is NOT renamed (only the reserved bare `main` is). Compile flags for Metal: `-I shaders -I mtlx/genslang` + `#define SKINNY_COMPUTE_PIPELINE 1`; **drop `-fvk-use-scalar-layout`** (Vulkan-only; Metal uses MSL layout natively — see §3).
  - **KEY BLOCKER (O2 — not a language issue, a build-ordering one):** `main_pass.slang` does NOT compile standalone. It needs the renderer's **runtime-generated** Slang artifacts present on the include path first, exactly as the Vulkan `slangc` path emits them before compiling: (1) `shaders/generated_materials.slang` (`evalSceneGraph` aggregator — empty-scene baseline via `emit_megakernel_aggregator([], GRAPH_BINDING_BASE)`), (2) per-graph `shaders/generated/*_graph.slang`, (3) `shaders/python_materials_dispatcher.slang` (`PythonMaterial`/`loadPythonMaterial`). The genslang closures (`skinny_skin_layered_bsdf_genslang` etc.) are checked-in under `src/skinny/mtlx/genslang/` and resolve via the second `-I`. ⇒ the Metal `ComputePipeline` must run the same emission sequence the Vulkan one does in `__init__` before linking (see §2.1 / new design decision D7).
- [x] 1.2 Reflect the compiled module: dump the MSL field offsets/size of the uniform block (`uniform FrameConstants fc`, binding 0) vs the Vulkan scalar layout. Record for §3.
  - **Outcome (measured via SlangPy reflection, Metal vs Vulkan devices):** `FrameConstants` is **592 B / 16-align on Metal (MSL)** vs **512 B / 4-align on Vulkan (scalar)**. The embedded `Camera` is **288 B (MSL)** vs **272 B (scalar)** — confirms the D3 estimate exactly. `float3` fields (`camera.position`, `focusPlaneOrigin`, `focusPlaneNormal`) are **16 B and 16-aligned on MSL** vs 12 B / 4-aligned scalar; that padding is the entire delta. `float2`/`uint2` fields (`zoomMin/Max`, `pickPixel`) are 8 B on both. The uniform block is a single global `uniform FrameConstants fc` at binding 0 (no separate `SkinParams`/`Light` uniform — skin/std params arrive via SSBOs). Reflection API: `program.layout.parameters` → the `fc` param's `type_layout` (`.size`, `.alignment`, `.fields[i].offset`).
