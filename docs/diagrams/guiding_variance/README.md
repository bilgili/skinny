# Guiding variance & efficiency harness — outputs

Generated by [`scripts/guiding_variance_sweep.py`](../../../scripts/guiding_variance_sweep.py)
(change `neural-guiding-variance-harness`). This directory holds the checked-in
sweep configuration and, after a run, the result tables + SVG plots that feed the
paper's renderer-side variance/efficiency claims.

## Files

| File | What |
|------|------|
| `config.json` | The checked-in declarative sweep matrix (scenes, cells, seeds, budgets, reference spp, resolution). The driver's built-in `default_config()` is the source of truth; this is its serialization for review/override via `--config`. |
| `RESULTS_<scene>.md` | Per-scene result table (equal-time variance, equal-variance spp/time, `1/(var·t)` efficiency) with a `% source:` provenance line, plus the skipped-cell list. *Produced by a run.* |
| `manifest_<scene>.json` | Machine-readable run record incl. the **reference hash** + witnessed convergence error (checked in for reproducibility). *Produced by a run.* |
| `<scene>_equal_time.svg` / `_efficiency.svg` / `_equal_variance.svg` / `_variance_vs_spp.svg` | The four SVG figures. *Produced by a run.* |

## Running

Needs the built **Python 3.13 venv** (MaterialX + Vulkan/Metal — see `CLAUDE.md`).
Native Metal needs no extra env; the Vulkan backend needs the SDK on the dylib
path. Metal `slangc` compiles are **serialized** (one cell at a time) and run
under `scripts/guarded_metal.sh` per the thermal rule.

```bash
# native Metal (Apple Silicon)
PYTHONPATH=src:tests scripts/guarded_metal.sh -- \
  ./bin/python3.13 scripts/guiding_variance_sweep.py \
    --backend metal --out-dir docs/diagrams/guiding_variance

# Vulkan / MoltenVK
export VULKAN_SDK=/Users/ahmetbilgili/VulkanSDK/1.4.341.1/macOS
export DYLD_LIBRARY_PATH=$VULKAN_SDK/lib
PYTHONPATH=src:tests ./bin/python3.13 scripts/guiding_variance_sweep.py \
  --backend vulkan --out-dir docs/diagrams/guiding_variance

# one-scene/two-cell low-spp smoke (task 5.2) — emits a table + a plot
... scripts/guiding_variance_sweep.py --backend metal --quick
```

Seeds are deterministic (each seed renders from a disjoint per-sample RNG stream,
`frame_index = seed × 1e6`), so a re-run with the same `config.json` reproduces
the scene set, reference, and numbers within Monte-Carlo tolerance.

## Methodology (MAX rigor)

- **Reference** per `(scene, integrator)`: a high-spp `{bsdf}` accumulation, gated
  by **two independent half-budget references agreeing** within 2 % before any
  cell is compared against it (a number against an unconverged reference is
  meaningless).
- **Variance**: per-pixel MSE vs the reference over the **linear-HDR accumulation
  image** (not the tonemapped output), aggregated as **mean ± spread over N
  independent seeds** — never a single run — with a firefly p99.9 percentile
  reported alongside the mean.
- **Metrics**: equal-time (variance at a fixed budget), equal-variance (spp +
  wallclock to a target variance, log-log inverted from the budget grid), and
  `1/(var·t)` efficiency — the metric identical to `ParametrizationResults §5`.
- The estimator is validated off-GPU (uniform image → ~0) in
  `tests/test_guiding_variance_metrics.py`.

## Coverage / degrade-gracefully

The matrix enumerates all axes but renders only the cells the build can produce.
Today the **chart** axis is fixed at `V1` (the shipped Lambert chart;
`renderer-chart-selection` not landed) and **temporal** at `off`
(`neural-temporal-conditioning` not landed). Cells requesting other charts /
temporal=on are **skipped and listed** in `RESULTS_<scene>.md` so the gap is
visible. Encoding (E0/E1/E3) and precision (fp32/fp16) are live.

## Paper wiring (cgf-paper task 2.5)

Each `RESULTS_<scene>.md` table carries a `% source:` provenance line giving the
exact driver invocation — the transcription hook for `cgf-paper`'s task 2.5 (the
`\Needed` renderer results / timings / renders). The four `<scene>_*.svg` figures
are the figure assets. *(The `cgf-paper` repository is not checked out under
`~/projects` in this environment, so the figures are produced and provenance-
tagged here for transcription rather than written into that repo directly.)*
